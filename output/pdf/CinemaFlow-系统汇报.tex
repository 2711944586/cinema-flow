\section{系统简介}\label{ux7cfbux7edfux7b80ux4ecb}}

\hypertarget{ux4e1aux52a1ux80ccux666f}{%
\subsection{业务背景}\label{ux4e1aux52a1ux80ccux666f}}

CinemaFlow
面向个人影迷、电影资料整理者和课程项目展示场景。普通表格或笔记软件可以记录片名和评分，但当电影数量增多后，用户会自然遇到几个问题：电影资料分散、导演与作品关系不清晰、观影计划和观影日志没有闭环、收藏与偏好难以统计、媒体图片
URL 质量无法统一检查。

CinemaFlow
将电影资料、导演档案、观影计划、观影日志、影评、推荐、偏好画像和片库审计整合到一个系统里。系统不只是展示电影卡片，而是围绕``电影资料长期维护''和``观影行为沉淀''建立完整业务链路。

\hypertarget{ux6570ux636eux5bf9ux8c61}{%
\subsection{数据对象}\label{ux6570ux636eux5bf9ux8c61}}

系统围绕以下业务对象展开：

\begin{itemize}
\tightlist
\item
  电影：片名、导演、类型、评分、时长、语言、简介、海报、背景图、预告片。
\item
  导演：姓名、国籍、出生年份、头像、活跃年代、风格、获奖、代表作。
\item
  用户电影状态：是否已看、是否收藏、个人评分、个人笔记、观看次数。
\item
  影评：作者、评分、标题、正文、点赞数。
\item
  待看片单：优先级、计划日期、观看场景、状态、备注。
\item
  观影日志：观看日期、地点、陪伴对象、会话评分、情绪标签、笔记。
\item
  媒体资源：电影海报、背景图、导演头像、提供方、检查时间。
\item
  最近浏览：用户进入详情页后的访问轨迹。
\item
  智能选片预设：时长、评分、类型、语言、是否包含已看、是否偏好收藏。
\end{itemize}

\hypertarget{ux67b6ux6784ux8bbeux8ba1ux8bf4ux660e}{%
\section{架构设计说明}\label{ux67b6ux6784ux8bbeux8ba1ux8bf4ux660e}}

\hypertarget{flask-ux5355ux4f53-bs-ux67b6ux6784ux7684ux7279ux70b9}{%
\subsection{Flask 单体 B/S
架构的特点}\label{flask-ux5355ux4f53-bs-ux67b6ux6784ux7684ux7279ux70b9}}

传统 Flask 单体 B/S 架构通常由 Flask
同时承担页面渲染、接口、静态资源、表单提交、业务逻辑和数据访问。

\begin{center}
\centering
\includegraphics[width=0.96\linewidth,height=0.56\textheight,keepaspectratio]{output/pdf/diagrams/report-diagram-02.png}
\captionof{figure}{Flask 单体 B/S 架构}
\end{center}

这种架构适合页面数量少、交互简单的系统。对于 CinemaFlow
这种页面数量多、局部状态多、搜索筛选频繁、需要命令面板和多种分析页面的系统，单体模板会让前端交互和后端数据逻辑紧密耦合。页面变化会影响后端模板，接口也不容易复用给其他客户端。

\hypertarget{ux524dux540eux7aefux5206ux79bbux67b6ux6784}{%
\subsection{前后端分离架构}\label{ux524dux540eux7aefux5206ux79bbux67b6ux6784}}

CinemaFlow 采用 Angular SPA + Flask REST API
的前后端分离结构。前端负责路由、组件、状态流和交互体验；后端负责 REST
API、数据校验、跨域策略和持久化；数据库负责保存电影资料和用户行为。

\begin{center}
\centering
\includegraphics[width=0.96\linewidth,height=0.56\textheight,keepaspectratio]{output/pdf/diagrams/report-diagram-03.png}
\captionof{figure}{前后端分离架构}
\end{center}

\hypertarget{ux67b6ux6784ux804cux8d23ux5bf9ux6bd4}{%
\subsection{架构职责对比}\label{ux67b6ux6784ux804cux8d23ux5bf9ux6bd4}}

\begin{longtable}[]{@{}lll@{}}
\toprule
维度 & Flask 单体 B/S & CinemaFlow 前后端分离 \\
\midrule
\endhead
页面渲染 & Flask 模板生成 HTML & Angular 组件在浏览器渲染 \\
页面路由 & 后端 route 控制页面跳转 & Angular Router 控制 SPA
页面和子路由 \\
数据交换 & 模板上下文或表单提交 & REST JSON \\
状态管理 & 依赖页面刷新和服务端上下文 &
RxJS、BehaviorSubject、combineLatest、LocalStorage \\
前端部署 & 依赖后端应用一起发布 & 前端构建产物可独立托管 \\
后端职责 & 页面、静态资源、接口混合 & API、校验、持久化、CORS \\
数据库演进 & 常和模板逻辑缠在一起 & API
合约稳定，数据库可在后端内部演进 \\
扩展客户端 & 复用困难 & REST API 可提供给移动端、小程序或其他客户端 \\
\bottomrule
\end{longtable}

\hypertarget{ux7cfbux7edfux5206ux5c42}{%
\subsection{系统分层}\label{ux7cfbux7edfux5206ux5c42}}

系统按职责拆成七个层次，各层之间通过清晰的数据对象和服务接口协作，避免页面组件直接处理持久化细节。

\begin{itemize}
\tightlist
\item
  表现层：对应 \texttt{pages/} 和
  \texttt{components/}，负责页面布局、用户交互、表格、卡片、图谱和表单展示。
\item
  路由层：对应 \texttt{app.routes.ts} 和
  \texttt{auth.guard.ts}，负责页面路由、详情页子路由和添加页权限守卫。
\item
  状态层：对应
  \texttt{MovieStateService}，聚合电影、最近浏览、影评、日志、计划等数据，形成页面
  ViewModel。
\item
  业务服务层：对应
  \texttt{MovieService}、\texttt{DirectorService}、\texttt{WatchLogService}
  等服务，负责实体管理、查询封装、API 同步和本地降级。
\item
  配置层：对应 \texttt{api.config.ts} 和
  \texttt{runtime-config.js}，负责解耦构建产物和 API 根路径。
\item
  后端 API 层：对应 Flask Blueprints，提供电影和导演 REST
  API，并负责请求校验、CORS 和持久化入口。
\item
  数据层：对应 JSON 演示持久层和关系型
  schema，保存电影、导演、用户行为、媒体资源和审计数据。
\end{itemize}

\hypertarget{ux6570ux636eux5e93ux8bbeux8ba1}{%
\section{数据库设计}\label{ux6570ux636eux5e93ux8bbeux8ba1}}

\hypertarget{ux8bbeux8ba1ux539fux5219}{%
\subsection{设计原则}\label{ux8bbeux8ba1ux539fux5219}}

CinemaFlow 的数据库采用关系型设计，核心原则如下：

\begin{itemize}
\tightlist
\item
  电影和导演属于主数据，字段稳定、复用频率高。
\item
  类型、演员、媒体资源拆成独立表，避免在电影表中堆叠多值字段。
\item
  收藏、已看、个人评分、笔记属于用户和电影之间的状态，因此放入
  \texttt{user\_movie\_states}。
\item
  影评、待看片单、观影日志属于用户围绕电影产生的业务事件，应独立建表。
\item
  情绪标签是观影日志的多值属性，使用 \texttt{watch\_log\_moods} 拆表。
\item
  最近浏览和审计日志属于行为轨迹，支持统计和问题追溯。
\item
  媒体资源表用于记录海报、背景图、头像等 URL，便于审计和替换。
\end{itemize}

\hypertarget{ux5173ux8054ux5173ux7cfbux8bf4ux660e}{%
\subsection{关联关系说明}\label{ux5173ux8054ux5173ux7cfbux8bf4ux660e}}

\begin{itemize}
\tightlist
\item
  \texttt{directors} 到 \texttt{movies}
  是一对多：一位导演可以对应多部电影。
\item
  \texttt{movies} 和 \texttt{genres} 是多对多：通过
  \texttt{movie\_genres} 支持一部电影同时属于科幻、动作、剧情等类型。
\item
  \texttt{movies} 和 \texttt{people} 是多对多：通过
  \texttt{movie\_people} 记录演员、角色名和排序。
\item

  \texttt{user\_movie\_states} 保存用户自己的收藏、已看、评分和笔记。

  是一对多：一条观影日志可被标注为治愈、热血、失眠夜等多个情绪。
\item
  \texttt{media\_assets} 使用 \texttt{entity\_type\ +\ entity\_id}
  支持电影、导演、人物等多种实体的媒体资源管理。
\item
  \texttt{audit\_logs} 使用 \texttt{entity\_type\ +\ entity\_id}
  记录不同实体的创建、更新和删除行为。
\end{itemize}

\hypertarget{ux6743ux9650ux7b56ux7565}{%
\subsection{权限策略}\label{ux6743ux9650ux7b56ux7565}}

系统权限边界采用``读公开、写受控''的设计：

\begin{itemize}
\tightlist
\item
  读接口：电影列表、电影详情、导演列表、导演详情、健康检查可公开访问。
\item
  写接口：新增、更新、删除电影和导演需要登录。
\item
  前端权限：\texttt{/add} 页面由 \texttt{authGuard}
  控制，未登录会跳回仪表盘并提示用户。
\item
  后端权限：正式数据库版本应在 POST、PUT、DELETE
  接口处校验用户身份和角色。
\item
  审计策略：关键写操作应记录到
  \texttt{audit\_logs}，保存操作人、实体、修改前和修改后的 JSON。
\end{itemize}

\hypertarget{ux5a92ux4f53-url-ux8bbeux8ba1}{%
\subsection{媒体 URL 设计}\label{ux5a92ux4f53-url-ux8bbeux8ba1}}

电影海报、背景图和导演头像使用真实远程 URL。系统会对图片 URL
进行规范化：

\begin{itemize}
\tightlist
\item
  TMDB 图片统一转换为稳定尺寸路径。
\item
  Wikimedia 图片统一走可追溯公开图片地址。
\item
  旧的随机占位图或不稳定图源不会作为最终展示依据。
\item
  图片加载失败时使用通用电影素材兜底，而不是生成伪造电影图。
\item
  预告片使用``片名 + 年份 + official trailer''的 YouTube
  搜索链接，保证和电影标题相关。
\end{itemize}

\subsection{关键设计取舍}\label{ux5173ux952eux8bbeux8ba1ux53d6ux820d}}

\begin{longtable}[]{@{}ll@{}}
\toprule
设计点 & 取舍说明 \\
\midrule
\endhead
前端本地状态与后端 API 并存 &
保证浏览体验稳定，同时保留向数据库持久化演进的接口边界 \\
图片 URL 真实化 &
避免随机占位图造成资料不可信，优先使用可追溯远程资源 \\
电影状态拆为用户状态表 &
多用户场景下，同一部电影的收藏、已看和评分应因人而异 \\
页面 ViewModel 服务化 &
避免页面模板直接拼装复杂统计，让组件更专注展示 \\
详情页使用子路由 & 让基本信息、演员表等内容可以独立扩展 \\
片库审计独立成页 & 把资料质量变成可见功能，而不是隐藏在代码逻辑中 \\
\bottomrule
\end{longtable}

\hypertarget{ux603bux7ed3}{%
\subsection{总结}\label{ux603bux7ed3}}

CinemaFlow
的核心价值在于把电影资料、用户观影行为和偏好分析连接起来。它不是单纯的电影列表，而是一个完整的私人电影资料馆系统。系统通过前后端分离架构保持页面交互和后端数据边界清晰，通过关系型数据库设计表达电影、导演、演员、用户行为和审计之间的关系，通过多页面功能覆盖浏览、记录、推荐、分析和维护的完整链路。
